# UI/UX 设计规范文档 - ReplyNow 配置助手 (ReplyNow Config GUI)

## 1. 视觉设计系统 (Design System)

为提供极致的高级质感与极简体验，软件采用深色暗黑美学（Sleek Dark Mode），核心视觉规范如下：

### 1.1 颜色与阴影 Token
```css
:root {
  --bg-deep: #0a0b10;          /* 软件底层底色，深邃太空黑 */
  --bg-surface: #12131a;       /* 卡片、面板前景色 */
  --bg-surface-hover: #1b1d28; /* 卡片悬浮色 */
  
  --color-primary: #6366f1;    /* 皇家靛蓝 Indigo */
  --color-secondary: #06b6d4;  /* 极光青 Cyan */
  --grad-accent: linear-gradient(135deg, var(--color-primary) 0%, var(--color-secondary) 100%);
  --grad-success: linear-gradient(135deg, #10b981 0%, #059669 100%);
  --grad-danger: linear-gradient(135deg, #ef4444 0%, #dc2626 100%);

  --text-primary: #f3f4f6;     /* 主文字 */
  --text-secondary: #9ca3af;   /* 辅助文字 */
  
  --border-color: rgba(255, 255, 255, 0.06);
  --border-focus: rgba(99, 102, 241, 0.4);
  --shadow-premium: 0 8px 32px 0 rgba(0, 0, 0, 0.37);
  --glass-blur: blur(12px);
}
```

## 2. 软件界面布局 (Layout Wireframe)

由于功能极大精简，界面专注于单页面极简操作，去除侧边栏和多余页签，窗口尺寸设定为精致的 `600px` (宽) x `450px` (高)。

```
+-----------------------------------------------------------------------------------+
|  ReplyNow 配置助手 (ReplyNow Config GUI)                               [-] [o] [x] |
+-----------------------------------------------------------------------------------+
|                                                                                   |
|   [● Codex 已就绪]                                                                 |
|                                                                                   |
|   请输入配置信息:                                                                 |
|                                                                                   |
|   API 地址 (API Base URL)                                                         |
|   [ https://api.replynow.cn:6688/v1                                            ]  |
|                                                                                   |
|   API 密钥 (API Key)                                                              |
|   [ ***************************************************************         (👁) ]  |
|                                                                                   |
|                                                                                   |
|   [ 测试连接 (Test) ]                           [ 一键保存并应用 (Save) ]          |
|                                                                                   |
|   -----------------------------------------------------------------------------   |
|   [ 状态反馈面板 ]                                                                 |
|   ● 连接成功! 延迟: 75ms  (或展示详细连接错误日志...)                                |
|                                                                                   |
+-----------------------------------------------------------------------------------+
```

### 2.1 界面元素说明
1.  **状态栏**：顶部显示本地 Codex CLI 目录环境检测状态（“已就绪”或“未检测到，点击一键初始化”）。
2.  **API Base URL 输入框**：默认自动填入 `https://api.replynow.cn:6688/v1`，允许点击并修改。
3.  **API Key 输入框**：掩码展示，右侧带有密码显隐切换小图标（👁）。
4.  **测试按钮**：点击触发 `test_connection` 并在下方的状态反馈面板实时输出。
5.  **保存按钮**：主色渐变，点击后执行保存及备份，并弹出 Toast 提示框。

## 3. 核心交互流程与动效 (User Flows & Animations)

### 3.1 动效设计细节
*   **连通性测试 (Pulse 绿圈脉冲)**：
    *   当连接测试成功时，绿色的状态圆点会触发一个向外扩散、逐渐淡出的多重波纹动画（Pulse Wave），给用户直观的成功反馈。
*   **错误状态抖动 (Error Shake)**：
    *   若测试连通失败（如网络超时或 API Key 错误），输入框的边框瞬间变为红色，并伴随 0.3s 的轻微左右抖动。
*   **加载等待 (Loading Spin)**：
    *   点击测试时，测试按钮文字切换为圆环 Loading Spinner，采用极光青渐变旋转。
